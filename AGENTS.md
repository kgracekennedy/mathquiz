# Repository Guidelines

## Project Structure & Module Organization
This repo is a React + Vite + PWA app for third-grade math practice. Use:
- `src/` for application code, quiz logic, and tests
- `public/` for install icons and static assets
- `.github/workflows/` for GitHub Pages deployment
- `requirements/` for product notes and scope decisions

Keep quiz generation, reporting, and UI flow separate so future skills like fractions can plug into the same session engine.
Commit product-memory docs to GitHub. In this repo, `requirements/requirements.md`, `requirements/version-history.md`, `AGENTS.md`, and `README.md` are part of the source of truth and should not live only in chat history.
Treat versions as explicit product milestones, not automatic counters. Do not increment a version number unless the user explicitly says to create a new version.
Do not add or edit entries in `requirements/version-history.md` unless the user explicitly asks for that update.

## Build, Test, and Development Commands
Use the scripts in `package.json`:
- `npm install` to install dependencies
- `npm run dev` to run the local Vite app
- `npm run build` to type-check and build production assets
- `npm test` to run Vitest
- `npm run lint` to run the TypeScript no-emit check

If dependencies change, regenerate and upload `package-lock.json`.

## Coding Style & Naming Conventions
Use 2-space indentation and keep files ASCII unless a file already needs something else. Prefer:
- `PascalCase` for React components
- `camelCase` for functions and variables
- `kebab-case` for route and asset names

Keep `vite.config.ts` on `vitest/config` because the file includes a `test` block.

## Testing Guidelines
Use Vitest and Testing Library. Keep test files in `src/**/*.test.ts` or `src/**/*.test.tsx`.

Production builds exclude test files through `tsconfig.app.json`, so do not move app logic into test-only modules.

## Deployment Rules
This repo deploys to GitHub Pages through a workflow in `.github/workflows/`.

For future repos:
- if a Pages deploy workflow already exists, update that file
- if no workflow exists, create `.github/workflows/deploy-pages.yml`

For this repo specifically:
- the current workflow file is `.github/workflows/main.yml`
- Vite `base` must match the repo name; this repo uses `/mathquiz/`
- keep `.npmrc` pinned to `https://registry.npmjs.org/`
- `dist/` is generated by CI and should not be uploaded manually

Do not upload or commit generated artifacts:
- `node_modules/`
- `dist/`
- `*.tsbuildinfo`
- `vite.config.js`
- `vite.config.d.ts`

## Change Checklist
When deployment or tooling changes, verify these together:
- `vite.config.ts`
- `.github/workflows/`
- `.npmrc`
- `package-lock.json`

If GitHub Actions shows missing React/JSX modules, check the install step first before changing app code.

When product behavior changes, update these docs together if needed:
- `requirements/requirements.md` for current product truth
- `requirements/version-history.md` only when the user explicitly wants to record or revise a version milestone
- `README.md` for current scope and run instructions
- `AGENTS.md` for repo and workflow rules
